## What list comprehensions solve

Building a list by iterating over a sequence, filtering items, and transforming them typically requires a multi-line for-loop with an if-statement and calls to `.append()`. List comprehensions collapse this pattern into a single readable expression that is both more concise and often faster.

## How it works

A list comprehension `[expr for item in iterable if condition]` is syntactic sugar that the Python compiler translates into a specialized loop with `LIST_APPEND` bytecodes — faster than manually calling `.append()` in a for-loop.

## Pseudocode

~~~python
# List comprehension
result = [x**2 for x in range(6) if x % 2 == 0]

# Equivalent for-loop
result = []
for x in range(6):
    if x % 2 == 0:
        result.append(x**2)
~~~

## Nested comprehensions

~~~python
# Nested loops
matrix = [[1, 2], [3, 4]]
flattened = [item for row in matrix for item in row]
# → [1, 2, 3, 4]

# Dict comprehension
squares = {x: x**2 for x in range(5)}
# → {0: 0, 1: 1, 2: 4, 3: 9, 4: 16}

# Set comprehension
evens_set = {x for x in range(10) if x % 2 == 0}
# → {0, 2, 4, 6, 8}
~~~

## Common pitfalls

- **Overly complex comprehensions**: nesting three or more loops or conditions inside a comprehension hurts readability — use a regular for-loop instead.
- **List vs generator**: a list comprehension builds the entire list in memory; use a generator expression `(expr for item in iterable)` for lazy evaluation.
- **Confusing assignment with walrus operator**: `[x := f(y) for y in items]` mutates the loop variable in the enclosing scope.
