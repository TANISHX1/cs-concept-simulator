## What closures solve

Functions often need to carry context — values computed earlier — into a later call. Without closures a function can only use its explicit parameters and global variables. Closures let a function bundle its body with the variables it references from the enclosing scope, making it self-contained.

## How they work

At the implementation level, a closure is a struct containing:
1. A pointer to the function's machine code.
2. Slots for each captured variable (or a pointer to a shared environment frame).

When the outer function returns, its stack frame is destroyed but the closure's environment is kept alive by the garbage collector or reference counting.

## Pseudocode

~~~javascript
function createCounter() {
    let count = 0;             // local variable
    return function() {        // closure
        count = count + 1;     // captures `count`
        return count;
    };
}

const counter = createCounter();
console.log(counter());        // 1
console.log(counter());        // 2
console.log(counter());        // 3
~~~

## Common pitfalls

- **Loop capture**: creating closures inside a loop that all reference the same mutable variable (classic `var` bug in JS).
- **Memory leak**: accidentally keeping a closure alive holds all captured variables in memory, preventing garbage collection.
- **Unexpected sharing**: multiple closures capturing the same mutable variable share mutations.
- **Performance**: capturing large objects prevents them from being freed until the closure itself is unreachable.
